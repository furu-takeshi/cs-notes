# Dictionaries

## 1. Dictionary Basics

*   **Key-Value Pairs**: Stores data as `{key: value}`.
*   **Unordered**: Access is by Key, not by index (Note: Ordered insertion is preserved in Python 3.7+).
*   **Mutable**: Can be changed after creation.
*   **Key Requirements**: Keys must be **unique** and **immutable** (e.g., strings, numbers, tuples). Values can be any data type.

## 2. Operations

*   **Accessing**: `D['key']` returns the value.
*   **Adding/Updating**: `D['new_key'] = new_value`
*   **Deleting**: `del D['key']`

## 3. Key Methods

| Method           | Description                                                        |
| :--------------- | :----------------------------------------------------------------- |
| `D.keys()`       | Returns a view object of all the keys.                             |
| `D.values()`     | Returns a view object of all the values.                           |
| `D.items()`      | Returns a view object of all key-value pairs (as tuples).          |
| `D.get(key)`     | Returns the value for key, or `None` if the key is not found (safer than `D[key]`). |
| `D.pop(key)`     | Removes the item with key and returns its value.                   |
